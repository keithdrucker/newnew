Enhance the ticketing system to standardize, categorize, and visualize Root Cause and Resolution data for continuous improvement and training insights.

Context:
Each ticket already includes Root Cause and Resolution fields. We want to:
- Standardize these into controlled categories
- Enable reporting and analytics
- Identify trends for training, system improvements, and process changes

--------------------------------------------------
DATA MODEL ENHANCEMENT
--------------------------------------------------

1. Add structured fields to tickets:

- root_cause_category (enum)
- resolution_category (enum)

2. Keep existing free-text fields:
- root_cause_notes
- resolution_notes

3. Ensure backward compatibility:
- Existing tickets without categories should default to:
  "Unknown / Not Categorized"

--------------------------------------------------
ROOT CAUSE CATEGORIES (ENUM)
--------------------------------------------------

Use the following controlled list:

- User Error / Training Gap
- Software Issue / Bug
- Configuration Issue
- Access / Permissions
- Network / Connectivity
- Hardware Failure
- Data Issue
- External Vendor Issue
- Security / Policy Block
- Unknown / Not Categorized

--------------------------------------------------
RESOLUTION CATEGORIES (ENUM)
--------------------------------------------------

Use the following controlled list:

- User Training / Guidance
- Configuration Change
- Software Fix / Patch
- Access Granted / Modified
- System Restart / Reset
- Hardware Replacement
- Escalated to Vendor
- Process Change
- Documentation Updated
- No Issue Found

UI / UX REQUIREMENTS

  - Root Cause Category
